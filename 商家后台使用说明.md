# 商家后台管理系统

## 功能概述

商家后台管理系统为咖啡商家提供了完整的店铺管理功能，包括商品管理、订单处理、评价查看和店铺设置。

## 访问方式

1. 使用商家账号登录系统（role 必须为 'merchant'）
2. 登录后点击右上角用户图标，系统会自动跳转到商家后台
3. 或直接访问：`/merchant/dashboard`

## 功能模块

### 1. 数据概览 (`/merchant/dashboard`)
- 查看店铺核心数据统计
  - 我的商品总数
  - 订单数量
  - 总收入
  - 待处理订单数
- 查看最近订单列表
- 查看热销商品排行
- 快捷操作入口

### 2. 我的商品 (`/merchant/products`)
- 商品列表展示（仅显示当前商家的商品）
- 添加新商品
  - 商品名称、分类、价格、库存
  - 商品描述
  - 多图上传
  - 上架/下架状态
- 编辑商品信息
- 删除商品
- 搜索和筛选功能

### 3. 订单管理 (`/merchant/orders`)
- 订单列表（仅显示包含当前商家商品的订单）
- 订单状态筛选
  - 待支付
  - 已支付
  - 已发货
  - 已完成
  - 已取消
- 查看订单详情
  - 订单信息
  - 收货地址
  - 商品明细
- 订单发货操作（已支付订单）

### 4. 评价管理 (`/merchant/reviews`)
- 查看商品评价列表（仅显示当前商家商品的评价）
- 按评分筛选（1-5星）
- 查看评价详情
- 搜索功能

### 5. 店铺设置 (`/merchant/settings`)
- **基本信息**
  - 店铺名称
  - 联系电话
  - 联系邮箱
  - 店铺地址
  - 店铺简介
- **账户安全**
  - 修改密码
- **通知设置**
  - 新订单通知
  - 评价通知
  - 库存预警
  - 系统消息

## 权限控制

- 只有 `role` 为 `merchant` 的用户才能访问商家后台
- 商家只能管理自己的商品和订单
- 路由守卫自动拦截未授权访问

## 技术特点

1. **独立的布局系统**
   - 咖啡色主题设计
   - 侧边栏导航
   - 响应式布局

2. **数据隔离**
   - 商家只能看到和管理自己的数据
   - 通过 `merchantId` 进行数据过滤

3. **实时数据**
   - 统计数据实时计算
   - 订单状态实时更新

4. **用户体验**
   - 加载状态提示
   - 操作确认对话框
   - 成功/失败消息提示
   - 图片预览功能

## 数据库要求

确保后端数据库中：
1. `users` 表有 `role` 字段，值为 'merchant'
2. `products` 表有 `merchantId` 字段，关联商家ID
3. `order_items` 表有 `merchantId` 字段（如果需要）

## 后端 API 要求

需要后端提供以下接口：
- `PUT /order/{orderId}/status` - 更新订单状态
- `GET /review` - 获取评价列表（支持分页）

## 使用流程

### 商家首次使用
1. 管理员创建商家账号（role='merchant'）
2. 商家登录系统
3. 进入商家后台
4. 完善店铺信息
5. 添加商品
6. 开始接收订单

### 日常操作
1. 查看数据概览，了解店铺运营情况
2. 处理新订单，及时发货
3. 管理商品库存
4. 查看和回复用户评价
5. 根据数据调整经营策略

## 注意事项

1. 商品图片上传需要后端支持
2. 订单发货后状态变更为 'shipped'
3. 评价数据通过商品ID关联到商家
4. 建议定期检查库存，避免缺货

## 与管理员后台的区别

| 功能 | 管理员后台 | 商家后台 |
|------|-----------|---------|
| 用户管理 | ✓ | ✗ |
| 所有商品管理 | ✓ | ✗ |
| 自己的商品管理 | ✗ | ✓ |
| 所有订单管理 | ✓ | ✗ |
| 相关订单管理 | ✗ | ✓ |
| 系统设置 | ✓ | ✗ |
| 店铺设置 | ✗ | ✓ |
